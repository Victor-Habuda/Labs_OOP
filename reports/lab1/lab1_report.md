## 1. Опис завдання
Згідно з номером залікової книжки (5103), визначено наступні параметри:
* **C5** (5103 % 5) = **3**: Операція — **XOR**.
* **C7** (5103 % 7) = **0**: Тип елементів — **double**.
* **C11** (5103 % 11) = **10**: Завдання — Виконати операцію XOR над елементами матриць.

**Мета:** Реалізувати алгоритм побітового виключного "АБО" (XOR) для матриць типу `double`, використовуючи перетворення в бітове представлення `long`.

---

## 2. Математичне обґрунтування
Операція XOR (`^`) в Java не застосовується безпосередньо до `double`. Програма виконує наступні кроки для кожного елемента:

1.  **Конвертація:** `Double.doubleToRawLongBits(value)` отримує 64-бітне представлення числа за стандартом IEEE 754.
2.  **Обчислення:** Виконується побітова операція $C = A \oplus B$.
3.  **Зворотна конвертація:** `Double.longBitsToDouble(bits)` повертає результат у формат `double`.

### Приклад розрахунку (Hex-вид):
Припустимо, ми робимо XOR для `12.0` та `0.0`:
* **12.0** у IEEE 754: `0x4028000000000000`
* **0.0** у IEEE 754: `0x0000000000000000`
* **Результат:** `0x4028000000000000` $\oplus$ `0x0000000000000000` = **12.0**
